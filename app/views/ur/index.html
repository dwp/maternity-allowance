Round 4: 10 November 2021

Prototype iteration 4

  • To break up the long page of claim summary information, we decided to use tabs to break the content into more manageable sections.
  • Tabs are an experimental pattern in the Design System, and require further research.
  • To further our understanding of the MAP rules and the information agents require for cases which fall into each rule group, we created 2 scenarios for testing in this round of research.
  • Scenario 1 – The claimant’s baby has been born between the 11th and 4th week. The claimant’s chosen MAP start date is too far in the future. The MAP starts the day after the baby is born.
  • Scenario 2 – The claimant’s employment ended before the 11th week. Their MAP starts on the 11th week.

  • We believe that grouping the information into manageable sections, displayed in tabs, will prevent the user from being overwhelmed.
  • We believe that grouping the information into manageable sections, displayed in tabs, will enable the agent to easily digest the information displayed.
  • We believe that the tab headings, which are all displayed above the fold, will reassure agents that all the information they need to see is available. They will not need to scroll to check that it is all there.
  • We will observe user behaviour through usability testing to see if this is the case.
' } }, { heading: { text: "Research notes" }, content: { html: '
  • Number of participants: 5

Round 5: 17 November 2021

Prototype iteration 5

  • We added (excluding title) to the Full name label, because some agents have entered a title in this field, which we don’t need to collect, and which is difficult for us to handle technically.

  • We believe that agents will be able to understand the status of a case quickly because of the tag in the blue banner.

Round 6: 24 November 2021

Prototype iteration 6

{% from "govuk/components/accordion/macro.njk" import govukAccordion %} {% from "govuk/components/accordion/macro.njk" import govukAccordion %} {{ govukAccordion({ id: "accordion-default", items: [ { heading: { text: "Design decisions" }, content: { html: '
  • We added a flag to make it clear which MAP rule applies to the case to reduce the amount of thinking an agent needs to do, and enable them to quickly recognise why a claimant’s preferred start date has not been allowed.
  • We moved the MAP rule information to the MAP dates tab, because agents in usability testing couldn’t remember where they had seen the rule, and expected it to be with the date information.
  • We changed the styling at the top of the page to remove the blue banner but keep important claimant information at the top, because agents in usability testing haven’t noticed the blue banner, and have missed information within it.
  • We moved the 13 weeks of earnings into flat content, because a lot of agents haven’t accessed the details drop-down without being prompted.
  • We have created 2 scenarios so we can cover more ground.
  • Scenario 1 is a 4-week non-flexible MAP with a Variable rate award. Their employment has been broken up, but they have only had 1 employer.
  • Scenario 2 is a Flexible MAP with a Standard rate award. Their employment dates cover 2 employers, and they have weekly earnings from 2 employers.
  • We know that claimants sometimes contact us to question which weekly earnings have been used, so we have included a breakdown of the earnings by employer, and a total weekly amount.
  • We have not included employer details in the Employment and Earnings tab because we cannot currently get this information from Finance GZ. We will explore with users what details they need here, and use this information to make additional requests to Finance GZ where appropriate.
' } }, { heading: { text: "Hypotheses" }, content: { html: '
  • We believe agents will need to see more information from the highest earning weeks, especially when the case is a variable rate.
  • We believe agents are more likely to notice the claimant details now they have been taken out of the blue banner, which we observed was not seen by most users.
  • We believe the MAP rule flag will reassure users that the system has accurately worked out the claimant’s start date, and they will be able to explain why, should a claimant call us.
  • We will test these hypotheses through observation and discussion in usability testing.
' } }, { heading: { text: "Research notes" }, content: { html: '
  • Number of participants: 5
  • The status tag on the claimant requested MAP start date received really positive feedback. It was quick and easy for agents to see whether that date has been used which is helpful, especially for answering queries.
  • The highest earning weeks flat content worked well, agents commented that the highest weeks had been used and they were actively looking at the information contained here.
  • Variable rate and multiple employers didn’t highlight any big issues. Both were expected to be tricky, but agents responded well.
  • Missing information highlighted this round were the employer name and whether the figure presented is gross/net income. We have heard these comments previously, but the description of the earnings figure appeared to be more important for the scenarios that were tested here.
  • There was still apprehension about clicking into the tabs, some completely missed them without prompting.
  • The white key details bar seems to have been more successful this round, we will continue to explore this.
' } } ] }) }}
