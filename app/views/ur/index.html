Round 1: 5 May 2022

Prototype iteration 1

{% from "govuk/components/accordion/macro.njk" import govukAccordion %} {% from "govuk/components/accordion/macro.njk" import govukAccordion %} {{ govukAccordion({ id: "accordion-default", items: [ { heading: { text: "Design decisions" }, content: { html: '
  • As we move into the Private Beta phase we have reorganised the prototype to group iterations tested in Alpha phase together. Private Beta iterations will sit in a new folder.
  • At the start of the journey we have created a simple landing page from which the agent will choose the type of activity they need to complete: Start a claim or Find a claim.
  • We were unable to find any established patterns from other agent-facing services, so have created something new.
  • We considered radio buttons but felt this created additional effort for the user. We also considered plain links on the page but this looked aesthetically odd. We have used Gov Blue and created a large target area for each option which we believe will be easy for agents to use as they progress through their workload.
  • We also considered having a default state for agents such as Start a claim, and an option for agents to move from that default state to Find a claim if they needed to, but this felt complicated, and there is no clear indication of what a default selection should be.
  • The data gather questions have been changed from conditional reveals to radio questions with date input on a following page. This removes Accessibility concerns raised around the conditional reveal.
  • We have changed the format of the pages to have a short H1 and a following question - we hope that this will help users to enter the correct dates on each page.
  • On the Summary we have changed our tags to be blue and lowercase to see if this aids readability and removes any potential bias or assumptions related to the colour of tags.
  • We have added the name of the employer as we can gather this from RTI. We have used the name registered with Companies House as this is what HMRC can provide to us.
  • As we have built a good level of trust among agents and a longer period of time has lapsed since we last tested with them, we want to explore whether we can remove some information from the Summary page and maintain that agent trust and confidence. The first thing we have decided to remove is the dates used for the Employment and Earnings tests.
  • Research has shown that agents expect to find a claim using the claimant’s National Insurance Number so we have used that for the Find a claim search.
  • At this time we have not included anything for a security check, as agents have indicated there may be enough information on the summary page to confirm the claimant’s identity through standard questions. We may need to amend this if additional ID&V requirements are exposed.
' } }, { heading: { text: "Hypotheses" }, content: { html: '
  • We believe the new H1 and subsequent question will help agents to enter the required dates in the right places.
  • We believe that we can remove the dates used for the Employment and Earnings tests because the highest earning weeks are detailed further down the page, and that the agent will trust the system has worked out the correct number of weeks for the Employment test.
  • NOTE: For scenarios where the Employment test is NOT met, we believe we will need to provide additional information such as the number of weeks they were employed, by whom, and when.
' } }, { heading: { text: "Research notes" }, content: { html: "

Number of participants:

" }, content: { html: '
  • Participants: 4
  • The landing page tested well for the first iteration, agents understood what they would use each option for and how to process the new claim in front of them.
  • Scenario 1 (date capture & interruption) the date last worked, and the MA start date questions were generally okay, but the maternity leave start date caused agents to make errors. Some self-corrected these errors the second time they saw the questions following the interruption.
  • Scenario 2 (find – multiple employers) the employer’s name didn\’t cause any issues. Agents commented on the addition of this as being something they don’t usually see or get now in legacy.
  • The earnings and employment test weeks were removed from the table, this did not seem to cause any problems for the agents and this information was not flagged as being missing.
